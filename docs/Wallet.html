new Wallet(options)

HD BIP-44/45 wallet

Parameters:
Name Type Description
options Object
Properties
Name Type Attributes Description
provider Provider <nullable>

Wallet Provider. If not present, no coins will be available.
master HDPrivateKey | HDPublicKey <nullable>

Master HD key. If not present, it will be generated.
addressMap Object <nullable>

Map of addresses to paths (internal).
witness Boolean <nullable>

Whether to use witness programs.
accountIndex Number <nullable>

The BIP44 account index (default=0).
receiveDepth Number <nullable>

The index of the next receiving address.
changeDepth Number <nullable>

The index of the next change address.
copayBIP45 Boolean <nullable>

Use copay-style BIP45 if bip45 derivation is used.
lookahead Number <nullable>

Amount of lookahead addresses (default=5).
type String <nullable>

Type of wallet (pubkeyhash, multisig) (default=pubkeyhash).
derivation String <nullable>

Derivation type (bip44, bip45) (default=bip44).
compressed Boolean <nullable>

Whether to use compressed public keys (default=true).
m Number <nullable>

m value for multisig.
n Number <nullable>

n value for multisig.
id String <nullable>

Wallet ID (used for storage) (default=account key "address").

fill(tx, optionsnullable)

Fill a transaction with inputs, estimate transaction size, calculate fee, and add a change output.

Parameters:
Name Type Attributes Description
tx MTX

Must be a mutable transaction.
options Object <nullable>
Properties
Name Type Attributes Description
selection String <nullable>

Coin selection priority. Can be age, random, or all. (default=age).
accurate Boolean

Whether to use "accurate" fee calculatation rather than rounding to the nearest kilobyte. See TX#getMinFee vs. TX#getMaxFee.
confirmed Boolean

Select only confirmed coins.
free Boolean

Do not apply a fee if the transaction priority is high enough to be considered free.
fee BN <nullable>

Use a hard fee rather than calculating one.
subtractFee Number | Boolean

Whether to subtract the fee from existing outputs rather than adding more inputs.
